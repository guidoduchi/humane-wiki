# Wiki Workflow

This page explains how to work with the HUMANE wiki without turning it into a pile of disconnected notes.

## Repository Boundary

This repository is for:

- canon,
- lore,
- worldbuilding,
- systems,
- continuity,
- chapter support,
- narrative-development reasoning.

The manuscript itself belongs elsewhere.

## Canon Promotion Flow

Use this flow when a new idea appears:

1. Draft the idea in the relevant page.
2. Mark uncertain details clearly.
3. Check contradiction against current canon.
4. Decide whether the idea is reader-facing, hidden canon, or future material.
5. Promote it into the canon page only when stable.

## Suggested Status Labels

At the top of pages, use one of these when helpful:

```text
Status: Canon
Status: Working Canon
Status: Draft
Status: Prototype
Status: Deprecated
```

## Suggested Spoiler Labels

```text
Spoiler Level: Reader-safe
Spoiler Level: Book 1
Spoiler Level: Full Canon
Spoiler Level: Future Material
```

## Editing Rule

Prefer small, meaningful updates over giant restructuring commits.

A good commit should answer:

> What became clearer?

## Review Rule

Before treating a page as stable, check:

- timeline consistency,
- naming consistency,
- character motivation,
- world logic,
- scientific plausibility,
- emotional consequence.
