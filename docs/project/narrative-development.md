# Narrative Development Guide

This wiki is not just a storage room for facts.

It is a working instrument for developing **HUMANE** into a stronger novel.

## Purpose

Use this wiki to make narrative decisions visible, traceable, and testable.

A good wiki page should help answer:

- What is true right now?
- Why does this matter to the story?
- What does the reader need to feel?
- What should remain hidden from the reader?
- What would break if this detail changed?

## The Central Standard

Every canon detail should serve at least one of these:

1. character pressure,
2. plot causality,
3. theme,
4. historical texture,
5. scientific plausibility,
6. emotional consequence.

If a detail is beautiful but serves nothing, keep it in a notes section until it earns its place.

## Narrative Principle

The world should not organize itself around the protagonist.

The protagonist moves through a living world that has its own hunger, politics, weather, memory, cruelty, tenderness, and momentum.

## Development Loop

Use this loop when building or revising material:

1. Define the canon fact.
2. Define the narrative function.
3. Define the emotional cost.
4. Define the reader-facing version.
5. Define what remains hidden.
6. Cross-check the timeline.
7. Cross-check character motivation.
8. Only then promote the material.

## Page Quality Checklist

A useful page should be:

- clear enough for a new collaborator,
- precise enough for continuity work,
- restrained enough to avoid over-explaining the novel,
- flexible enough to evolve,
- human enough to remind us why the facts matter.
